## 🪙 서비스 소개

- How to start Finance? 재테크를 시작해보려고 하시나요? 저희는 'HoWoo Finance'라는 웹 플랫폼을 통해 예적금 상품 비교 및 상품 추천 서비스를 제공합니다. HoWoo Finance는 팀원인 "Ho"jun + Young"Woo"의 합성어로 How를 구상해서 만든 단어입니다. 이제 막 재테크를 시작하여 금융 상품에 대해 알고 싶은 이용자를 위해 제작하였으며, 금융서비스가 처음이신 분들은 저희가 어렵지 않도록 도움을 드리겠습니다.

- 'How Finance'란, "재테크 방법, 금융의 방법"이라는 뜻을 가지고 있으며, 저희 웹 플랫폼인 "HoWoo Finance"가 이용자를 위해 상품을 비교하고, 지금 당장 가입할 수 있는 상품 조회 서비스를 제공하오니 한 번 이용하시길 권합니다.

## 🦾주요 기능

- 메인페이지
    - 사이트 첫 화면. 주요 기능에 대한 간략한 안내를 포함
    - 필수 기능 및 사용자의 관심을 끌 수 있는 콘텐츠(상품 조회 및 추천, 내 주변 근처 은행 검색 지도, 커뮤니티 게시판, 마이페이지 등의 기능)를 네비게이션 및 퀵 바(quick bar)로 제공
- 회원 관리
    - 회원 가입을 통해 개인 정보를 등록하고, 로그인을 통해 상품 가입, 게시글 작성 및 수정 기능 접근 가능

    - 로그인한 사용자의 경우, 마이페이지에서 가입한 상품 조회 및 금리 비교 그래프 조회 가능

    - 다양한 은행들의 예적금 상품의 최신 금리 정보를 제공

- 커뮤니티 게시
    - HoWoo Finance 이용자들 간의 소통과 정보 공유를 위한 공간. 이용자들은 금융에 관련된 질문이나 정보를 게시하고, 다른 회원들과 의견을 나눌 수 있음.
- 각종 편의 기능
    - 근처 은행 검색 : 사용자의 위치 정보 및 검색어를 활용, 주변에 위치한 은행을 찾아줌.
    - 환율조회 및 환율 계산 : 사용자가 현재 환율 정보를 조회하고 환율 계산을 할 수 있도록 도움.
    - 퀵메뉴 제공 : 사용자 다른 페이지로 이동 시 손 쉽게 이동 가능

## ⚙금융 상품 추천 AI ChatBot에 대한 기술적 설명

저희 HoWoo Bot은 ***이용자가 요구하는 질문에 최적의 조건에 부합하는 금융 상품을 추천***하는 서비스 입니다.

1. 이용자가 ChatBot과의 채팅창에 나이, 취향, 투자성향 등의 정보를 입력하면, 해당 정보는 axios를 통해 Django 서버로 전송되어 API에 전달됩니다.
2. 서버에서는 전달받은 정보를 기반으로 Prompt를 활용하여 데이터베이스에서 필터링을 수행합니다. AI ChatBot은 입력된 정보에 대해 이용자에게 가장 좋은 효율을 뽐내는 가입 상품을 필터링합니다.
3. 보통은 최고 우대금리를 기준으로 추천해주며, 이용자가 조건을 자세하게 할 수록 조건에 부합하는 상품을 제공합니다.

이렇게 구현된 AI ChatBot인 Howoo Bot은 사용자의 정보와 요구하는 조건에 대해 다양한 상품들 중 이용자에게 알맞는 최적의 상품을 금융상품을 추천해주는 기능을 수행합니다.

### 김호준 🏅

약 4개월이 넘는 기간동안 열심히 배웠지만서도 막상 하나의 서비스를 구현하려니, 많이 헷갈리고 구현이 막힐 때마다 힘들었던 것 같습니다. 하지만 팀원과 함께 노력하고 에러가 나도 계속 시도하다보니 언젠가는 해결이 되면서 그때부터는 많은 걱정없이 노력만해서 완성할 수 있었던 것 같습니다. 또한 첫 프로젝트인만큼 팀원과 정말 열심히 했지만, 한편으론 좀만 더 하면 더 잘할 수 있었을 것 같은 느낌에 약간의 아쉬움도 많이 남는 프로젝트인 것 같습니다. 하지만 이번 첫 프로젝트는 맛보기일 뿐 진짜는 2학기 프로젝트라고 생각하고, 더 열심히 해야할 것 같습니다. 시작할 때만 해도 1주일 동안 가능한 프로젝트인지 의심이 가기도 했지만, 끝나갈 때쯤에 같은 반 친구들 모두 거의 끝나가는 상황인 것 보니 정말 모두 엄청 노력했다는 것이 느껴지고 많이 공부할 수 있었던 기회였던 것 같습니다. 또한 방학 때 제가 무엇을 해야할 지가 슬슬 눈에 보이기 시작하여 좋은 경험이였습니다. 앞으로도 쭉 힘듦이 찾아와도 노력만 한다면 시간이 해결해줄 것이라 믿고 열심히 노력하겠습니다.


### 김영우 🏅

Back, Front를 동시에 다루면서 데이터를 요청하고 DB에 저장해서 화면에 출력하는 전체 과정을 경험 하였습니다. 400 Bad Request, 401 Unauthorized 등 다양한 에러를 직접 확인하고 코드를 고쳐나가는 과정이 힘들고 어려웠지만, 에러를 해결할 때 마다 목표를 한 단계씩 이루어나가는 느낌이 들어 좋았습니다. 지난 1학기 동안 배운 모든 것을 복습하고 실제로 적용해 볼 수 있었던 유익한 프로젝트였습니다.
